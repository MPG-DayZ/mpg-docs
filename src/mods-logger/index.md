## Описание

**Серверный мод ведения логов** для DayZ-модов. Предоставляет единый класс логгера, который моды-потребители
используют через фабричный паттерн - каждый мод получает собственный файловый лог с буферизацией, уровнями и
архивацией, без дублирования кода логгера в каждом моде.

## Основные возможности

- Фабричный паттерн: каждый мод создаёт свой экземпляр с изолированными настройками
- 6 уровней логирования (TRACE → FATAL) с фильтрацией по минимальному уровню
- Буферизация записи (flush каждые 0.5с) для снижения нагрузки на диск
- Немедленный flush для ERROR/FATAL - критичные сообщения не теряются при крахе
- Авто-архивация лога при рестарте + ротация (до 500 архивов, настраивается)
- Двойной вывод: файл + серверная консоль (RPT) с отдельным порогом
- Два временных штампа: реальное время + ticktime для замеров интервалов
- Кэширование временного префикса по секунде - оптимизация при интенсивном логировании
- Опциональность: потребитель работает и без мода (fallback на `Print`)
- Изоляция модов: каждый мод пишет в свой файл, логи не смешиваются
- Без конфликтов: класс определён один раз, моды совместимы друг с другом
- Хелпер `FormatPosition(vector)` для логирования координат

## Установка

::: warning Это серверный мод!
Клиенту мод не нужен - только серверу.
:::

Установить мод можно любым, привычным для вас способом, скачав его со Steam.

https://steamcommunity.com/sharedfiles/filedetails/?id=3755713051

```
-serverMod=@MPG_ModsLogger;@YourMod
```

## Использование

Мод используется модами-потребителями через прекомпилятор `#ifdef MPG_ModsLogger`. При отсутствии мода
логирование деградирует до стандартного `Print()` в RPT.
