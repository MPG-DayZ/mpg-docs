# Правила использования Api Клиента

## Кеширование крайне важно

> [!IMPORTANT]
>
> Т.к. приложение является бутылочным горлышком в плане потока данных, крайне важна реализация кеширования ответов на уровне приложения.

Реализация кеширования не входит в состав пакета, поэтому оно должно быть реализовано в рамках проекта и в соответствии с требованиями по кешированию на проекте.

## Это обёртка над ofetch

По сути api клиент представляет собой простую обёртку над [ofetch](https://github.com/unjs/ofetch) через автоматическую генерацию с помощью пакета [@webui/ofetch-client](https://gitlab.letoile.tech/webui/packages/ofetch-client).

Пакет не содержит никакой дополнительной логики по обработке URL или обработке ошибок.

Пакет предназначен для упрощения использования эндпоинтов и поставке типов для этих эндпоинтов, что бы упростить разработку и сопровождение кода.

Для полноценной интеграции пакета требуется реализовать свой обработчик ошибок с учётом требований проекта и добавить свою логику.
Сделать это достаточно просто т.к. по сути это обёртка над ofetch.

## Разделяй SSR и клиент

Для проектов на vike необходимо передавать абсолютный URL для получения данных, если обращение идёт с сервера (SSR), в тоже время для SPA это противопоказано т.к. api живёт на том же домене, что и приложение.

```ts twoslash
// +data.ts
// @noErrors
import { appConfig } from '@server/config/appConfig';
import { strapiCommon } from '@webui/vi-api-strapi';

export const getStrapiData = (query: string) => {
  return strapiCommon(query, {
    // Необходимо задать baseUrl для API
    baseURL: `${appConfig.APP_PROXY_URL}`,
  });
};
```

```vue
<script setup>
import { search } from '@webui/vi-api-searcher';

await search(/*body*/);
</script>
```
