# Быстрый старт

Эта страница - для тех, кто ставит мод впервые. Здесь по шагам показано, как за пару минут сделать первую рабочую точку
спавна. Подробное описание каждого параметра - на страницах [Конфиг мода](mod-config.md) и
[Конфиг точки](point-config.md).

## Шаг 1. Установите мод

::: warning Это серверный мод!
Его нужно ставить только на сервер, на клиент он не нужен (отладочные моды для админов - отдельно, см. ниже).
:::

Установите мод любым привычным способом со Steam:

https://steamcommunity.com/sharedfiles/filedetails/?id=3120779051

## Шаг 2. Запустите сервер - мод создаст конфиги

При первом запуске сервера мод сам создаст конфигурационные файлы в папке профиля сервера:

```
<профиль сервера>/
└─ MPG_Spawner/
   ├─ Config.json              ← общие настройки мода
   └─ Points/
      ├─ Point_Example1.json   ← примеры точек
      └─ Point_Example2.json
```

Папка профиля сервера - это обычно путь, заданный параметром `-profiles=` при запуске сервера. Мод добавляет в неё
подпапку `MPG_Spawner/`.

В этих файлах и есть вся настройка. Примеры точек можно отредактировать или удалить - это просто заготовки.

## Шаг 3. Сделайте первую точку

Откройте любой файл из папки `Points/` (например `Point_Example1.json`) и приведите его к минимальному виду.
Для рабочей точки достаточно всего **шести** параметров - остальное можно не трогать:

```
[
  {
    "pointId": 1,
    "triggerPosition": "4246.0 0.0 5586.7",
    "triggerRadius": "40.0",
    "spawnPositions": ["4246.0 0.0 5586.7"],
    "spawnMin": 2,
    "spawnMax": 5,
    "spawnList": ["Animal_UrsusArctos"]
  }
]
```

Что это значит:

| Параметр                | Что делает                                                          |
|-------------------------|---------------------------------------------------------------------|
| `pointId`               | Уникальный номер точки (не повторяется нигде больше)                |
| `triggerPosition`       | Координаты центра зоны на карте (X Z Y). `0.0` по высоте - на земле |
| `triggerRadius`         | Радиус зоны в метрах - где игрок включает срабатывание              |
| `spawnPositions`        | Точки, где появится живность (можно несколько)                      |
| `spawnMin` / `spawnMax` | Сколько живности заспавнится за одно срабатывание                   |
| `spawnList`             | Класснеймы живности (можно с настройками через `\|`)                |

::: tip Где взять координаты?
Координаты можно снять в DayZ-Editor, либо командой в игре. Формат - три числа через пробел: `X высота Y`.
:::

Сохраните файл, перезапустите сервер - точка готова. Игрок, зайдя в зону, запустит спавн.

## Шаг 4. Добавьте себя в админы (для отладки)

Чтобы видеть визуализацию зон и подписи над живностью, добавьте свой SteamID в список админов в `Config.json`:

```
"admins": [
  "76561198022444444"
]
```

::: tip Где взять свой SteamID?
Используйте любой сервис вроде steamid.io - нужно именно длинное число (SteamID64), начинающееся с `7656...`.
:::

## Шаг 5. Включите отладку

В точке включите визуализацию зоны и подписи над живностью:

```
"showVisualisation": 1,
"showEntityLabels": 1
```

- `showVisualisation` рисует полупрозрачную фигуру зоны - нужен мод
  [@MPG_Spawner_Debug](https://steamcommunity.com/sharedfiles/filedetails/?edit=true&id=3755477157)).
- `showEntityLabels` рисует над каждой заспавненной живностью подпись вида `1 · #3` (видят только админы).

Так вы сразу увидите, правильно ли стоит зона и где появляется живность.

Для перезагрузки конфига мода без рестарта сервера используйте
мод [@MPG_SpawnerConfigReload](https://steamcommunity.com/sharedfiles/filedetails/?id=3303128154) - это сэкономит уйму времени!

## Шаг 6. Смотрите логи

Если что-то не работает - загляните в логи:

```
<профиль сервера>/MPG_Spawner/Logs/MPG_Spawner_Log.log
```

Старые логи автоматически складываются в архив `Logs/LogArchives/`.

::: tip Ничего нет в логах?
Включите детальный лог для конкретной точки - `"isLogsEnabled": 1`. Тогда точка начнёт подробно писать, что происходит
(вход/выход игрока, решения о спавне, ошибки). Подробнее - в разделе [isLogsEnabled](point-config.md#islogsenabled).
:::
