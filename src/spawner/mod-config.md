# Конфиг мода

Конфиг мода находится в файле `MPG_Spawner/Config.json` внутри папки с профилем сервера.

::: tip Перезагрузка конфига без рестарта сервера
Используйте мод [MPG_SpawnerConfigReload](https://steamcommunity.com/sharedfiles/filedetails/?id=3303128154) для того, что бы не перезапускать сервер каждый раз, когда нужно изменить конфиги. 

Мод позволяет перезапускать конфиги отдельных триггеров или всегомода без перезагрузки сервера.
:::

## configVersion

`configVersion` - Версия конфига, служебное поле, не нужно его менять. Оно обновляется само при обновлении мода, если в
этом есть необходимость.

Актуальная версия конфига — `6`. При переходе на новую версию мода старые конфиги обновляются автоматически.

## documentation

`documentation` - Cлужебное поле, содержит ссылку на документацию по моду для вашего удобства.

## isModDisabled

`isModDisabled` - Полное отключение мода целиком.

::: warning Внимание
Параметр инверсный: `true` означает, что мод **выключен**. Что бы мод работал, оставьте `false`.
:::

Пример: `"isModDisabled": false,`

## logLevel <Badge type="tip" text="2.0" title="Добавлено в версии 2.0" />

`logLevel` - Глобальный уровень логирования для всего мода. Числовой параметр.

Заменяет удалённый параметр `isDebugEnabled` — теперь детализацией логов управляет одно число, а не флаг "всё или
ничего".

Доступные значения:

| Значение | Уровень | Что пишется                                                              |
|----------|---------|--------------------------------------------------------------------------|
| `1`      | TRACE   | Максимальная детализация: каждый шаг очистки, изменения состояния и т.п. |
| `2`      | DEBUG   | Решения потока: будет/не будет спавн, счётчики, переходы состояния       |
| `3`      | INFO    | Ключевые события: создание точки, вход/выход, спавн, победа              |
| `4`      | WARN    | Предупреждения: некорректные данные точки, аномалии спавна               |
| `5`      | ERROR   | Ошибки: объект не создан, не удалось удалить и т.п.                      |
| `6`      | FATAL   | Только критические ошибки                                                |

Значение по умолчанию: `3` (INFO).

::: info Как это работает вместе с [isLogsEnabled](point-config.md#islogsenabled) точки

Логи фильтруются в **две ступени**, и обе должны пропустить сообщение:

1. **Точка** (`isLogsEnabled`): если `false`, точка глушит всё ниже WARN. При этом WARN, ERROR, FATAL пишутся всегда.
2. **Мод** (`logLevel`): глобально отсекает всё ниже указанного уровня.

Важно: `logLevel` **не может** "разблокировать" то, что заглушила точка, и наоборот — точка не может разблокировать то,
что
отсёк глобальный `logLevel`.

:::

::: tip Рекомендации

- Обычная работа сервера — `3` (INFO).
- Нужно "тихо" — поставьте `6` (FATAL).
- Включить детальный лог для конкретной точки можно параметром [isLogsEnabled](point-config.md#islogsenabled), не
  поднимая
  `logLevel` для всего мода.

:::

Примеры:

```
Обычная работа (по умолчанию):
"logLevel": 3,

Почти полная тишина — только критические ошибки:
"logLevel": 6,
```

## pointsConfigs

`pointsConfigs` - Массив с названиями json файлов с конфигами точек в папке Points из которых будет загружена информация
для создания
триггеров.

Пример:

```
"pointsConfigs": [
  "Point_Example1",
  "Point_Example2"
]
```

## admins <Badge type="warning" text="1.4" title="Добавлено в версии 1.4" />

`admins` - Массив с админами.

Указывается steamId тех, кому разрешено использовать административные функции. 

На данный момент административные функции это отображение визуализации и перезагрузка конфига.

Пример:

```
"admins": [
  "76561198022444444",
  "76561198022555555"
]
```
